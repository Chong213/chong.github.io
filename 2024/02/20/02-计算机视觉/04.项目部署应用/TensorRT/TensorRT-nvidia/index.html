TensorRT-NVIDIA(基于8.2.3)

简介

TensorRT 文档、C++ API文档、Python API文档、TensorRT 下载、本教程配套代码及更多样例

  1. 用于高效实现已训练好的深度学习模型的推理过程的SDK
  2. 内含推理优化器和运行环境
  3. 使DL模型能以更高吞吐量和更低的延迟运行
  4. 有C++和python的API，完全等价可以混用

工作

  • 构建期（推理优化器）
    模型解析/建立 加载ONNX等其他格式的模型/使用原生API搭建模型
    计算图优化 横向层融合（Conv），纵向融合（Conv+add+ReLU），……
    节点消除 去除无用层，节点变换（Pad，Slice，Concat，Shffle），……
    多精度支持 FP32/FP16/INT8/TF32（可能插入reformat节点）
    优选kernel/format 硬件有关优化
    导入plugin 实现自定义操作
    显存优化 显存池复用
  • 运行期（运行时环境）
    运行时环境 对象生命期管理，内存显存管理，异常处理
    序列化反序列化 推理引擎保存为文件或从文件中加载

如何迁移部署已完成模型

  1. 使用框架自带TRT接口（TF-TRT，Torch-TensorRT）

    部署在原框架中

  2. 使用Parser（TF/Torch/…->ONNX->TensorRT）

    ONNX通用性好，便于调整模型，兼顾效率性能

  3. 使用TensorRT原生API搭建网络

    性能最优，惊喜网络控制，兼容性最好

基本流程

  • 构建阶段
    1. 建立Logger（日志记录器）
    2. 建立Builder（网络元数据）和BuilderConfig（网络元数据的选项）
    3. 创建Network（计算图内容）
    4. 生成SerializeNetwork（网络的TRT内部表示）
  • 运行阶段
    1. 建立Engine
    2. 创建Context
    3. Buffer准备（Host端和Device端）
    4. Buffer拷贝 Host to Device
    5. 执行推理（Execute）
    6. Buffer拷贝 Device to Host
    7. 善后工作

Logger（日志记录器）

  • 可选参数：VERBOSE、INFO、WARNING、ERROR、INTERNAL_ERROR，产生不同等级的日志，详->简（网络优化过程和网络优化结果的相关信息）

Builder引擎构建器

  • 新版本仅作为构建引擎 的入口，不在用于设置相关网络属性

  • 常用成员
    builder.max_batch_size = 256 指定最大Batch Size（Static Shape模式下使用）

  • Dynamic Shape 模式必须用builerConfig及相关API

BuilderConfig网络属性选项

  • 常用成员
    config.max_workspace_size = 1 << 30 指定构建期可用显存（单位：Byte）
    config.flag = … 设置标志位：如1 << int(trt.BuilerFlag.FP16), 1<< int(trt.BuilerFlag.INT8), …
    config.int8_calibrator = … 指定calibrator
    config.set_static_sources（限制算法实现）/set_timing_cache（节约构建时间）/algorithm-selector（精确控制节点算法）/… 等等

Network网络具体构造

  • 常见参数：

    1 << int(tensorrt.NetworkDefinitionCreationFlag.EXPLICIT_BATCH)，使用Explicit Batch模式

  • 常用方法：

    • network.add_input(‘oneTensor’, trt.float32, (3, 4, 5)) --> 标记网络输入标量
    • convLayer = network.add_convolution_nd(XXX) --> 添加各种网络层
    • network.mark_output(convLayer.get_output(0)) --> 添加网络输出张量

  • 常用获取网络新消息的成员：

    • network.name / network.num_layers / network.num_outputs
    • network.has_implicit_batch_dimension / network.has_explicit_precision
Explicit Batch模式 v.s. Implicit Batch模式
  1. TensorRT主流network构建方法，Implicit Batch模式builer.create_network(0)仅用作后向兼容
  2. 所有张量显式包含Batch维度、比Implicit Batch模式多一维
  3. 需要使用builder.create_network(1 << int(tensorrt.NetworkDefinitionCreationflag.EXPLICIT_BATCH))
  4. Explicit Batch模式能做、Implicit Batch模式不能做的事举例：
    • Layer Normalization
    • Reshape / transpose / Reduce over batch dimension
    • Dynamic shape 模式
    • Loop结构
    • 一些Layer的高级用法（如：ShuffleLayer.set_input）
  5. 从ONNX导入的模型也默认使用Explicit Batch模式
Dynamic shape 模式
  • 适用于输入张量形状在推理时才决定网络
  • 除了Batch维，其他维度也可以推理时决定
  • 需要Explicit Batch模式
  • 需要Optimazation Profile帮助网络优化
  • 需用context_binding-shape绑定实际输入数据形状
